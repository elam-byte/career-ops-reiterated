# Mode: coverletter — Cover Letter Generation

Generates a professional, ATS-optimised cover letter as a `.docx` file (opens in LibreOffice and Microsoft Word).

---

## Step 1 — Gather inputs

You need three things before writing. If any is missing, ask:

1. **JD** — job description text or URL. If URL, fetch with Playwright → WebFetch fallback.
2. **Generated (tailored) CV** — the already-tailored `.tex` file for this application.
   - Look first in `output/{company-slug}-{role-slug}/cv-{slug}.tex` (generated by `modes/pdf.md`)
   - If not found, ask the user: "Has the CV PDF been generated yet? If yes, tell me the company and role slug so I can find it. If not, run `/career-ops-reiterated pdf` first, then come back."
   - **Do NOT fall back to the raw template.** The cover letter must be based on the tailored CV so that both documents are consistent and reference the same framing and keywords.
3. **Company name and role title** — extract from JD if not provided.
4. **Hiring manager name** — scan the JD or company page for a named contact. If found, use it in the salutation. If not, use generic.

Also read `config/profile.yml` for candidate name, email, phone, location, LinkedIn.

## Step 3 — Extract from JD

Before writing, identify:
- Top 10–15 ATS keywords (exact phrases from JD requirements / responsibilities)
- Primary role dimension (what they are most hiring for)
- 3 most prominent JD requirements to map to the 3 achievement bullets
- 1–2 specific things about this company that are non-generic (product names, mission, known challenge, recent news) — for the "My particular value" section
- Seniority level expected

---

## Step 4 — Write the cover letter

### Format rules

- Tone: Formal, confident, concise — suitable for European corporate culture
- No generic phrases ("I am writing to apply for…", "I am passionate about…", "I believe I would be a great fit")
- No storytelling gimmicks, no filler
- No repetition of CV content — interpret and connect it to the role
- **No `--` or `—` (em dashes) in body text** — use `,` instead for mid-sentence pauses or appositions
- **Target: ~370 words total** (not counting header blocks and salutation)
- ATS keywords from Step 3 distributed naturally throughout
- Feels personalised and intentional — a recruiter should feel this was written for this specific role and company

### Text output format (MANDATORY — the generator script parses this)

The body text passed to the script must use these markers exactly:

| Element | Format |
|---------|--------|
| Normal body paragraph | plain text, one paragraph per line |
| Section heading | `**Heading text:**` — entire line wrapped in `**`, on its own line |
| Bridge sentence | plain text line immediately after a section heading line |
| Bullet with bold prefix | `- **Bold heading:** rest of body text` |
| Blank paragraph separator | empty line between blocks |

**Do NOT include** the salutation, sign-off, or signature in the body text — the script adds those automatically from separate flags.

Example body output:
```
Intro paragraph text here connecting who you are to the role.

**Why I can fill this role from day one:**
Your requirements for X, Y, and Z align exactly with my daily work at ZF:
- **Enterprise Architecture:** Defined E2E target architectures for an €80M portfolio; improved efficiency by 8-12%.
- **Cross-functional Delivery:** Shipped three Azure/Power Platform products at 99.95% availability; €5-10M/year savings.
- **Scalability & Risk:** Minimised systemic risks; 6-7 releases/year at 90% success via CI/CD and SLOs.

**My particular value for {Company}:**
Company-specific paragraph here.

**My contribution from day one:**
Certs + bridge + direct contribution statement.
```

---

### Structure

**— Header block (do not count in word count) —**

```
{Candidate full name}
{Address}
Phone: {phone}  |  Email: {email}

{Hiring manager name (if known)}
{Company name}
{Company address if known}
{City}, {date}
```

**— Subject line —**
```
Application as {Role Title} | {Job ID if available}
```
- EN: `Application as {Role}`
- DE: `Bewerbung als {Rolle} | {Job-ID}`

**— Salutation —**
- EN: `Dear {Ms./Mr. Lastname},` or `Dear Hiring Team,` if no name
- DE: `Sehr geehrte Frau/Herr {Nachname},` or `Sehr geehrte Damen und Herren,`

---

**Paragraph 1 — Introduction (2–3 sentences, no heading)**

- State who you are: current role + company + years of experience + primary domain
- Immediately connect to what the company is looking for — use their exact role title and a key JD phrase
- Show you understand their context from the first line
- Do NOT start with "I am writing to apply"

*Example pattern:* "As [current role] at [company], I [have been doing X relevant to their JD] for [N years]. Your search for a [exact JD role title] to [primary JD goal] matches exactly my profile with [X] years of [domain]."

---

**Section heading: "Why I can fill this role from day one:" (adapt wording to role)**

Role-adaptive heading variants:
- Architect role → `Why I can fill this role from day one:`
- Manager / EM role → `Why I can lead this team from day one:`
- Project / Portfolio role → `Why I can drive this programme from day one:`
- DE → `Warum ich die Position ab dem ersten Tag optimal ausfüllen kann:`

**Bridge sentence (1 sentence):** State that your JD requirements and daily work at ZF are fully aligned — use specific JD language.

**3 bullet points — each with a bold heading:**

Each bullet must:
1. Start with `**{JD dimension}:**` (bold in docx) — taken directly from the JD's top requirements; use the JD's exact vocabulary
2. The body: one tight sentence — real achievement + KPI from `cv.md` + connected to the JD theme
3. Be 1.5–2 lines maximum — compress if longer
4. Cover 3 distinct JD requirements — no two bullets on the same theme

Map bullets in order of JD prominence (most important first).

*Example pattern:*
- `**Enterprise Architecture Management:** Defined E2E target architectures for an €80M digital portfolio aligned with long-term business goals; increased governance efficiency by 8–12% through OKRs, guardrails, and stage-gates.`
- `**Cross-functional Orchestration:** Delivered three market-facing digital products on Azure/Power Platform (99.95% availability); generated €5–10M/year in savings through API-first integration.`
- `**Systemic Risk & Scalability:** Minimised scalability and security risks; shipped 6–7 releases/year at 90% success rate via CI/CD, SLOs, and observability; Azure Data Lake governance across 8–12 domains (Data-SLO 99.9%).`

---

**Section heading: "My particular value for {Company}:" (adapt to company)**

Role-adaptive heading variants:
- EN: `My particular value for {Company}:`
- DE: `Mein besonderer Mehrwert für {Company}:`

**1 paragraph (3–4 sentences):**
- Reference something specific about the company — product names, mission, known challenge, or market context found in Step 3 research (never generic praise)
- Connect it to a real, specific aspect of your experience (a role, a project, a domain you worked in)
- Show you understand their world, not just the job title
- If no specific company info is available, use the role's domain or challenge instead

---

**Section heading: "My contribution from day one:" (or adapt to role)**

Role-adaptive heading variants:
- EN: `My contribution from day one:`
- DE: `Mein Beitrag ab Tag eins:`

**1 paragraph (4–6 sentences):**
- Synthesise your most relevant certifications and cross-cutting experience: Azure Solution Architect Expert, PMP, MBA, iSAQB CPSA — mention only those most relevant to the JD
- Name the specific bridge you bring: between technical depth and business/stakeholder layer, or between corporate strategy and engineering delivery
- Add one sentence on the specific domain or technical area from the JD where you can contribute immediately, grounded in a concrete skill or past project from `cv.md`
- Add one sentence showing awareness of the team or organisation context (e.g. scale, maturity, cross-functional dynamics) and how your background maps to it
- Add one direct contribution statement: one sentence on the concrete, immediate impact you will deliver in this role (e.g. "I will bring X to your team from day one", grounded in the JD's primary need)
- Forward-looking — express clear interest in next steps without sounding desperate

---

**Closing line (1 sentence, no heading):**
- EN: Express interest in a personal conversation to discuss how you can contribute
- DE: `Ich freue mich darauf, in einem persönlichen Gespräch zu erläutern, wie ich [Company] bei [specific goal from JD] unterstützen kann.`

**Closing salutation:**
- EN: `Best regards,` / `Yours sincerely,`
- DE: `Mit freundlichen Grüßen`

**Signature:** Full name on new line.

---

### German cover letter specifics

If writing in German:
- Use formal `Sie` throughout
- Include job ID in subject line if available: `Bewerbung als {Rolle} | {Job-ID}`
- Heading wording:
  - `Warum ich die Position ab dem ersten Tag optimal ausfüllen kann:`
  - `Mein besonderer Mehrwert für {Unternehmen}:`
  - `Mein Beitrag ab Tag eins:`
- Bullet bold headings in German using JD vocabulary
- Closing: `Mit freundlichen Grüßen`

## Step 5 — Generate the .docx file

After writing the cover letter, call the generator with the structured flags below. Pass **only the body text** (intro + sections + bullets) via `--text` or stdin — do NOT include the salutation, sign-off, or signature; the script handles those.

```bash
python cv/generate_coverletter.py \
  --company "{Company}" \
  --role "{Role}" \
  --lang {en|de} \
  --subject "{Application as Role | Job ID}" \
  --salutation "{Dear Ms. Lastname,}" \
  --hiring-manager "{Ms. Firstname Lastname}" \
  --company-address "{Company Name\nDepartment\nStreet\nCity}" \
  --text "{body text with **headings** and - **bullets:**}"
```

Or pipe the body via stdin:
```bash
cat <<'EOF' | python cv/generate_coverletter.py \
  --company "{Company}" --role "{Role}" --lang {en|de} \
  --subject "..." --salutation "..." --hiring-manager "..." --company-address "..."
{body text}
EOF
```

- `--hiring-manager`: name shown bold in the recipient block (omit if unknown)
- `--company-address`: include the company name as the first line, then department, street, city (e.g. `"Siemens AG\nHR Department\nMain St 1\n80333 Munich"`). The script does NOT add the company name automatically — it must be in this field.
- Sign-off (`Mit freundlichen Grüßen` / `Best regards,`) and full name are added automatically

**Output:** `output/{company-slug}-{role-slug}/coverletter-{company-slug}-{role-slug}.docx`
