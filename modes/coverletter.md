# Mode: coverletter — Cover Letter Generation

Generates a professional, ATS-optimised cover letter as a `.docx` file (opens in LibreOffice and Microsoft Word).

---

## Step 1 — Gather inputs

You need three things before writing. If any is missing, ask:

1. **JD** — job description text or URL. If URL, fetch with Playwright → WebFetch fallback.
2. **Generated (tailored) CV** — the already-tailored `.tex` file for this application.
   - Look first in `output/{company-slug}-{role-slug}/cv-{slug}.tex` (generated by `modes/pdf.md`)
   - If not found, ask the user: "Has the CV PDF been generated yet? If yes, tell me the company and role slug so I can find it. If not, run `/career-ops-reiterated pdf` first, then come back."
   - **Do NOT fall back to the raw template.** The cover letter must be based on the tailored CV — the version already customised to this specific JD — so that both documents are consistent and reference the same framing and keywords.
3. **Company name and role title** — extract from JD if not provided.

Also read `config/profile.yml` for candidate name, email, phone, location, LinkedIn.

---

## Step 2 — Detect language

Same rule as `modes/pdf.md`:

| JD language | German requirement | Cover letter language |
|-------------|-------------------|-----------------------|
| German | Any | German |
| English | Mandatory (C1/C2, required, fließend) | German |
| English | Optional / nice-to-have | English |
| English | Not mentioned | English |

**State the detected language before writing.**

---

## Step 3 — Extract from JD

Before writing, identify:
- Top 5–8 ATS keywords (exact phrases from JD requirements / responsibilities)
- Primary role dimension (what they are most hiring for)
- 1–2 specific things about this company that are non-generic (mission, product, known challenge, recent news)
- Seniority level expected

---

## Step 4 — Write the cover letter

Use the following prompt and rules exactly:

### Writing rules

- Tone: Formal, confident, concise — suitable for German/European corporate culture
- No generic phrases ("I am writing to apply for…", "I am passionate about…", "I believe I would be a great fit")
- No storytelling gimmicks, no humor
- No repetition of CV content — interpret and connect it to the role
- Each paragraph has one clear purpose
- Feels personalized and intentional — a recruiter should feel this was written for this specific role
- 300–400 words total
- ATS keywords from Step 3 distributed naturally throughout

### Structure (4 paragraphs)

**Paragraph 1 — Value proposition (3–4 sentences)**
- Who you are: title + years of experience + primary domain
- What you bring: the single most relevant thing you offer this specific role
- Use the JD's own language — if they say "enterprise architecture" use that phrase, not "system design"
- Do NOT start with "I am writing to apply"

**Paragraph 2 — Most relevant experience with measurable results (4–5 sentences)**
- Pick the 1–2 achievements from the CV that best match the JD's primary requirement
- Lead with the outcome/metric, then explain what you did
- Use exact ATS keywords from the JD naturally embedded in sentences
- Read metrics from the CV template exactly — never invent numbers

**Paragraph 3 — Alignment with company and role (3–4 sentences)**
- Reference something specific about the company (from Step 3 research — not generic praise)
- Connect it to a real aspect of your experience
- Show you understand their context, not just the job title
- If no specific company info is available from the JD, use the role's domain/challenge instead

**Paragraph 4 — Professional closing (2–3 sentences)**
- Express clear interest in next steps
- Forward-looking, not desperate
- No "Thank you for your time and consideration" — too generic
- End with your full name on a new line

### German cover letter specifics

If writing in German:
- Use formal `Sie` throughout
- Standard German business letter opening (no "Sehr geehrte Damen und Herren" unless no name available — try to use hiring manager name from JD)
- Include subject line: `Bewerbung als {Rolle} — {Ihr Name}`
- Closing: `Mit freundlichen Grüßen`
- Same 4-paragraph structure and word count applies

---

## Step 5 — Generate the .docx file

After writing the cover letter text, call the generator script:

```bash
python cv/generate_coverletter.py \
  --company "{Company}" \
  --role "{Role}" \
  --lang {en|de} \
  --text "{cover letter text}"
```

Or pass via stdin:
```bash
echo "{cover letter text}" | python cv/generate_coverletter.py \
  --company "{Company}" \
  --role "{Role}" \
  --lang {en|de}
```

**Output:** `output/{company-slug}-{role-slug}/coverletter-{company-slug}-{role-slug}.docx`
